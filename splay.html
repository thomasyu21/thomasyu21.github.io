Splay Trees

Splay Trees were first invented by Daniel Sleator and Robert Tarjan in 1985. Splay trees build upon a binary search tree, with the addition of one operation called splaying.

So, lets focus on binary search trees first.

Binary Search Trees

Terminology

  • Node: an individual item in the tree
  • Key: the value of a node
  • Root: the top of the tree (the first node accessed in a search)
  • Edge: the link between two nodes
  • Parent: a branch to any other node
  • Child: a node with a link to a parent node
  • Siblings: nodes with the same parent
  • Leaf: a node with no children
  • Subtree: a section of the overall tree

How it works

Binary search trees follow two rules. First, each parent is limited to a max of two children. Second, the key of the left child must always be less than the key of parent and the key of the right child must always be greater than the key of the parent.

Heaps

Heaps are another type of data structure that makes use of a tree. However, unlike the binary search tree, it does not have the same relationship between parent and children nodes. Instead, the key of the parent is always greater than both children (max-heap) or less than both children (min-heap). This allows us to access the greatest or smallest value in the tree the fastest as it will be at the root node.

But what if you wanted quick access to nodes that didn't have the greatest or smallest key in the tree? That's where splay trees can help.
